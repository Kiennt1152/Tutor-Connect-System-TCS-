Entity State Specification

Status transition definitions for core entities

👤 User Account Status

Active

Normal operation - can login, transact

Locked

Temporary blocked - 5 failed attempts

Inactive

Disabled or expired account

💼 Job Posting Status

Draft

Work in progress - not public

Active

Live recruitment - visible to tutors

Closed

Position filled - cannot reopen

User Status Flow

Active → Locked → Inactive

5 fails / Admin disable / Long lock

Job Posting Flow

Draft → Active → Closed

Publish / Filled or Expired

Business Rules 1

Data & Workflow Rules

GB-01

Irreversible State Transition

Some status changes can only proceed forward, never backward. This ensures data integrity and prevents accidental reversals.

Class Flow (Example):

Created → Matched → In Progress → Completed

✗ Cannot reverse once completed

GB-04

Data Uniqueness Constraint

Unique fields ensure data integrity across the entire system.

  • Email (UNIQUE, NOT NULL)
  • Phone Number (UNIQUE, NOT NULL)
  • Tutor License Number
  • Center License
  • Class Code (auto-generated)
  • Transaction ID (UUID)

Duplicate handling: "Email already registered"

Risks & Mitigations (1/2)

Project risk assessment and response plan

ID Risk Level Mitigation
R-01 Facebook API changes HIGH Manual import backup, webhook redundancy
R-02 Data privacy violation HIGH GDPR/PDP compliance, encrypt all PII
R-03 Payment gateway downtime HIGH Multi-provider, retry mechanism
R-04 Scope creep MEDIUM Change Control Board, MoSCoW
R-05 Team member leave MEDIUM Documentation, code review
R-06 Database performance MEDIUM Index optimization, query audit
R-07 Third-party service failure LOW Circuit breaker, fallback UI
R-08 Security breach attempt LOW OWASP compliance, pentest

R-01: Facebook API Changes

  • Monitor API changelog regularly
  • Quick adaptation sprint (2-3 days)
  • Manual CSV import fallback

R-02: Data Privacy Violation

  • Encrypt all PII at rest & transit
  • Role-based data access
  • Privacy impact assessment

R-03: Payment Gateway Downtime

  • Backup: Stripe + PayOS
  • Queue retry for transactions
  • Clear user messaging
